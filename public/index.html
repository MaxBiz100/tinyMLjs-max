tinyMLjs

A set of hardware independent web-page, client-side (No Internet required) programming tools to help bring Machine Learning Proof of Concept's to more students, by using the power of the web with javascript-tensorflowJS and the simplicity of the C/C++ microcontroller serial print command combined together using Google's WebSerial that works on any desktop (or laptop) computer. Note: webSerial using polyfill works on a few Android phones such as some of the Google Pixel phones

name="basicInstructions">Basic instructions

  • Hardware independent proof of concept. Use a sensor or two and get a Machine Learning Keras model working with an actuator on any microcontroller board.
  • Once you know your model works, slow down and record your data into a CSV file for uploading to a professional cloud Machine Learning service like edgeImpulse.com or or Arduino Cloud or just stay client-side with tinyMLjs and build another model.
  • If staying with tinyMLjs Upload the CSV files to a model designer, train the model and save your tensorflowJS saved model
  • If staying with tinyMLjs load the saved model onto a webpage with webSerial connection to your actuator and test how well your =model works in real world settings.
  • or use the conversion program either downloaded to online that converts the TensorflowJS saved model into tensorflowLite format and then use xxd -i to convert the binary file into a c-header file that can be installed onto your microcontroller using your IDE or the Arduino IDE.
